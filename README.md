# 每日任务日程 — 功能与技术规划

## 项目概述

一个纯前端的每日任务管理网页，以单个 HTML 文件交付，浏览器直接打开即可使用。默认显示**日视图**，支持周视图查看整周待办，核心交互是**新建任务 → 完成任务 → 打勾**。数据存储在浏览器本地。

---

## 核心功能

### 1. 视图

- **日视图（默认）**：打开即显示今天的待办列表，顶部显示日期和星期，可左右翻页切换日期
- **周视图**：7 天横向排列，每天下方列出当天任务卡片（标题 + 优先级标签 + 完成状态），可快速浏览整周
- "回到今天" 快捷按钮

### 2. 每日任务管理

- 添加任务：输入框 + 回车或点击按钮快速添加
- 编辑任务：点击任务文字可修改标题、优先级、备注
- 删除任务：每条任务右侧有删除按钮
- **打勾完成**：每条任务左侧有 checkbox，勾选后标题加删除线并变灰，移到列表底部（已完成区）
- 取消打勾：点击已完成的勾选框可撤销完成状态
- 任务排序：未完成任务按优先级排序，已完成任务按完成时间倒序

### 3. 任务字段

| 字段 | 说明 |
|------|------|
| 标题 | 必填，任务的文字描述 |
| 优先级 | 必填，高 / 中 / 低 三档（颜色区分） |
| 备注 | 可选，补充说明文字 |
| 日期 | 必填，任务所属日期 |
| 已完成 | checkbox 状态 |

### 4. 优先级系统

- 三档优先级：🔴 高 / 🟡 中 / 🟢 低
- 添加任务时可选择优先级（默认中）
- 列表中任务按优先级排序：高 → 中 → 低
- 每条任务左侧有优先级颜色条，一目了然

### 5. 数据持久化

- 使用 `localStorage` 存储所有任务数据
- 页面加载时自动读取，任务变更时自动保存
- JSON 数据结构：

```json
{
  "tasks": [
    {
      "id": "uuid",
      "title": "完成项目报告",
      "date": "2026-05-28",
      "priority": "high",
      "note": "明天开会要用",
      "completed": false,
      "completedAt": null,
      "createdAt": "2026-05-28T08:00:00.000Z"
    }
  ]
}
```

### 6. 周视图统计

- 周视图每天卡片上显示：未完成数 / 总任务数
- 本周完成率统计

### 7. 响应式设计

- 桌面端（≥768px）：日视图左侧日期导航 + 右侧任务列表布局；周视图 7 列
- 移动端（<768px）：日视图全屏任务列表；周视图改为可横滑的单列卡片

## 技术栈

| 层面 | 选择 | 理由 |
|------|------|------|
| 结构 | HTML5 | 语义化标签 |
| 样式 | 原生 CSS（CSS Grid + Flexbox + 自定义属性） | 零依赖，现代浏览器完全支持 |
| 逻辑 | 原生 JavaScript（ES6+） | 零依赖，无需打包 |
| 存储 | `localStorage` | 数据持久化 |
| ID 生成 | `crypto.randomUUID()` | 浏览器原生 UUID |
| 日期处理 | 原生 `Date` + `Intl.DateTimeFormat` | 无需第三方库 |

**不引入任何第三方库或框架**，单文件即可离线运行。

## 实现步骤

1. **HTML 骨架** — 顶部导航栏、日视图任务列表、周视图网格、添加任务栏
2. **CSS 样式** — 任务卡片、优先级颜色、checkbox 动画、完成态样式、响应式
3. **数据层** — 任务 CRUD、localStorage 读写、优先级排序
4. **日视图逻辑** — 按日期筛选任务、渲染列表、添加/编辑/删除/打勾
5. **周视图逻辑** — 计算本周日期范围、按天分组渲染任务卡片
6. **视图切换与翻页** — 日/周视图切换、前后翻页、回到今天
7. **完成统计** — 本周/今日完成率
